# 《选佛谱》美术与交互动画总流程

## 一、核心原则

《选佛谱》不是普通掷骰走格游戏，而是：

> 两枚六字轮相一掷，玩家在十五门二百二十位之间升沉流转。

因此，美术资产和交互动画必须围绕四个动作组织：

```text
观图 → 掷轮 → 升沉 → 读谱
```

所有美术资产都不应孤立存在，而应被串成一次完整的“行棋仪式”。

## 二、三层视觉系统

游戏界面分为三层。

### 1. 十五门总览层

作用：告诉玩家自己处在整个须弥山升沉图的哪里。

资产：

```text
15门总览背景图
15门平台/层级标签
当前门高亮
已历门轨迹
可达门提示
升沉方向光效
```

交互：

- 玩家刚进入游戏时看到总览。
- 起手前，总览停在第一门附近。
- 行棋跨门时，镜头从当前门移动到目标门。
- 如果下降，镜头向下滑动并带墨色拖尾。
- 如果上升，镜头向上滑动并带金色流光。
- 如果横超净土，镜头不走常规上下路径，而以横向莲光桥过渡。

### 2. 当前门局部棋盘层

作用：真正进行走棋和点击位次。

资产：

```text
每门无文字2.5D背景图
门内节点台座
门内路径线
当前位高亮
已历位标记
可达位高亮
不行震动提示
```

交互：

- 默认只显示当前门位次，不显示220位。
- 当前位高亮。
- 玩家点任一位次可打开详情。
- 掷轮后目标位发光，再移动棋子。
- 走出本门时切换到总览层，完成跨门镜头移动。

### 3. 当前位详情层

作用：解释玩家“为什么会到这里”。

资产：

```text
谱曰卡片
原文/白话切换
轮相结果卡
升沉原因条
组合标签
```

交互：

- 点击节点时从底部弹出。
- 掷轮后自动显示本次结果摘要。
- 玩家可展开原文、白话、轮相表。
- 默认只显示摘要，避免手机屏幕被文字淹没。

## 三、一次完整回合流程

每一回合分 7 步。

### Step 0：静候

界面状态：

```text
当前位发光
两枚轮相静置
当前位详情卡收起或半展开
可达位可选择显示/隐藏
```

动画：

- 当前位有轻微呼吸光。
- 背景云气、水面、灯火轻微循环。
- 轮相有微弱阴影和木纹高光。

### Step 1：玩家按下“掷轮”

界面状态：

```text
按钮按下
轮相启动
地图轻微暗下
当前位仍保留高亮
```

动画：

- 两枚六字轮相旋转、翻滚。
- 周围出现淡淡金墨涟漪。
- 按钮从“掷轮”变为“轮转中”。
- 不需要过度刺激，应像仪轨，不像赌场。

### Step 2：轮相揭示

界面状态：

```text
显示二字组合，例如「阿佛」
组合标签放大一次
```

动画：

- 两枚轮相停定。
- 两个字从轮面上浮现到组合标签。
- 组合标签短暂发光。

音效建议：

```text
木轮轻响
小磬声
纸面轻震
```

### Step 3：查当前位轮相表

界面状态：

```text
当前位节点发光
详情卡显示“谱曰”摘要
```

动画：

- 当前位到目标位之间出现一条预备光线。
- 如果目标未知或不行，则当前位轻微震动。
- 如果是赠掷，则出现小型“赠掷”印章。

### Step 4：升沉判定

根据结果分为六类：

```text
上升：金线向上
下降：墨红线向下
同层轮转：淡墨环线
不行：节点震动，墨点散开
赠掷：朱砂印章
成佛：圆光展开
横超净土：莲光桥横向出现
```

这一步是游戏的情绪核心。

### Step 5：棋子移动 / 镜头移动

同门移动：

- 棋子沿门内路径移动。
- 路径线短暂发光。
- 目标节点变为当前位。

跨门移动：

- 当前门缩小为总览层。
- 镜头滑向目标门。
- 到达目标门后进入该门局部棋盘。

下降时：

```text
背景略暗
墨色拖尾
轻微坠落感
```

上升时：

```text
金线牵引
云气上卷
目标门发光
```

横超净土时：

```text
不沿普通路径移动
莲光桥横向展开
画面短暂切入净土色调
```

### Step 6：结果落定

界面状态：

```text
新当前位高亮
旧当前位变为已历位
日志增加一条
详情卡显示本次结果
```

动画：

- 新节点落光。
- 旧节点留下小墨点或脚印。
- 底部日志卷轴新增一行。

### Step 7：读谱

玩家可以：

```text
继续掷轮
点击详情看原文
展开白话解释
查看本位轮相表
回到15门总览
```

这一步让游戏从“走格子”变成“读谱修观”。

## 四、美术资产如何串联

### 资产 A：15门总览图

用途：跨门导航、全局升沉感。

使用位置：

```text
OverviewBoard.jsx
```

状态：

```text
normal
current
visited
reachable
crossing-up
crossing-down
pureland-cross
win
```

### 资产 B：每门背景图

用途：当前门局部棋盘。

路径建议：

```text
web/public/art/gate01/gate01-bg-2-5d.webp
web/public/art/gate02/gate02-bg-2-5d.webp
...
```

注意：背景图必须无文字，节点文字由 React 渲染。

### 资产 C：节点组件

用途：220位通用节点。

不建议为220位各画一张图，而是做通用节点样式：

```text
普通节点
当前节点
已历节点
可达节点
上升目标
下降目标
不行节点
终局节点
```

### 资产 D：轮相资产

用途：玩家掷轮。

包含：

```text
六字轮相静态图
六面字图：那、謨、阿、彌、陀、佛
滚动阴影
停定高光
组合标签
```

轮相应像仪轨木器，不像赌场骰子。

### 资产 E：动效光线

用途：表达升沉。

包含：

```text
上升金线
下降墨红线
轮转淡墨环线
横超莲光桥
不行墨点散开
成佛圆光
```

这些可以用 CSS/SVG/Canvas 实现，不一定要全部做成图片。

### 资产 F：谱曰卡片

用途：显示原文、白话、原因。

包含：

```text
底部半屏卡
全屏读谱卡
轮相表紧凑卡
朱砂组合标签
```

## 五、手机端推荐主流程

### 初次进入游戏

```text
首页 / 游戏模式
↓
15门总览缩略出现
↓
镜头落到第一门
↓
显示第一门局部棋盘
↓
按钮：起手掷轮
```

### 起手掷轮

```text
轮相滚动
↓
出现组合
↓
落入第一门某位
↓
该位高亮
↓
底部卡片显示“起手 · 发始因地门 · 某位”
```

### 常规行棋

```text
玩家掷轮
↓
轮相停定
↓
当前位查表
↓
显示升/沉/不行/横超
↓
棋子移动或震动
↓
详情卡解释
```

### 跨门

```text
门内地图缩小
↓
切到15门总览
↓
镜头沿升沉方向移动
↓
目标门发光
↓
进入目标门局部地图
```

### 终局

```text
掷出终局组合
↓
全图安静
↓
顶部圆光打开
↓
显示圆极果位
↓
日志收束为最终行迹
```

## 六、动画节奏建议

整体节奏要克制，不要像抽卡游戏。

推荐时长：

```text
轮相滚动：800–1200ms
组合揭示：250ms
路径预亮：250ms
棋子移动：500–900ms
跨门镜头：800–1200ms
结果卡出现：250ms
成佛圆光：1200–1800ms
```

交互原则：

```text
可跳过
不阻塞太久
减少闪烁
多用云、墨、金线、纸面动效
少用爆炸、粒子雨、强震动
```

## 七、技术实现建议

### React 状态层

游戏已有这些核心状态：

```text
pos 当前位
path 行棋轨迹
dice 当前轮相
last 最近结果
won 是否终局
picked 被点击的位次
```

建议增加：

```text
viewMode: overview | gate | detail
transition: idle | rolling | reveal | resolving | moving | settled
activeGateId
targetGateId
targetPosId
```

### 组件层

建议组件结构：

```text
Game.jsx
  OverviewBoard.jsx      15门总览
  GateBoard.jsx          当前门棋盘
  WheelRoller.jsx        六字轮相
  MoveResult.jsx         本次结果
  PositionDetail.jsx     原文白话详情
  JourneyLog.jsx         行棋记录
  MobileRollDock.jsx     手机底部操作条
```

### 数据层

建议数据结构：

```text
board-layout.overview.json
board-layout.gate01.json
board-layout.gate02.json
...
```

每门 JSON 负责：

```text
门信息
背景图路径
位次坐标
节点分组
路径建议
安全区
移动视觉提示
```

## 八、资产制作顺序

不要一次画完所有资产。建议顺序：

### 阶段 1：可玩原型

```text
第一门局部棋盘
15门总览低精度图
通用节点样式
轮相动画
结果卡片
```

### 阶段 2：美术统一

```text
15门总览精修
第一门背景精修
第二门、第三门试做
统一色彩、光线、节点比例
```

### 阶段 3：逐门扩展

```text
补齐其余门背景
处理别教52位特殊布局
补充净土横超、圆极果位终局动效
```

### 阶段 4：体验打磨

```text
跨门镜头
升沉动效
详情卡阅读体验
性能优化
图片压缩
低端手机降级模式
```

## 九、性能注意事项

手机端不能堆太多大图和动画。

建议：

```text
每门背景 webp，宽 1080 或 1440
总览背景单独压缩
节点用 CSS，不用大图片
路径用 SVG
光效用 CSS 渐变
长动画可关闭
低端手机关闭背景动态云雾
```

## 十、最终体验目标

玩家每一回合应感到：

```text
我掷的是轮相，不是骰子。
我走的是升沉，不是普通格子。
我看的不是随机结果，而是谱曰因果。
我在十五门中流转，不是在一张死棋盘上移动。
```

这就是《选佛谱》的交互核心。
